## Melhores práticas de acessibilidade de UI

- **HTML Semântico**: Use elementos HTML apropriados (nav, main, button, etc.) que transmitam significado para tecnologias assistivas
- **Navegação por Teclado**: Garanta que todos os elementos interativos sejam acessíveis via teclado com indicadores de foco visíveis
- **Contraste de Cor**: Mantenha taxas de contraste suficientes (4.5:1 para texto normal) e não dependa apenas de cor para transmitir informações
- **Texto Alternativo**: Forneça alt text descritivo para imagens e labels significativos para todos os inputs de formulário
- **Teste com Leitor de Tela**: Teste e verifique se todas as views são acessíveis em dispositivos de leitura de tela
- **ARIA Quando Necessário**: Use atributos ARIA para melhorar componentes complexos quando HTML semântico não for suficiente
- **Estrutura de Cabeçalho Lógica**: Use níveis de cabeçalho (h1-h6) na ordem correta para criar um esboço de documento claro
- **Gerenciamento de Foco**: Gerencie o foco apropriadamente em conteúdo dinâmico, modais e aplicações de página única (SPA)
