## Melhores práticas de estilo de código

- **Convenções de Nomenclatura Consistentes**: Estabeleça e siga convenções de nomenclatura para variáveis, funções, classes e arquivos em toda a base de código
- **Formatação Automatizada**: Mantenha um estilo de código consistente (indentação, quebras de linha, etc.) e utilize formatadores automáticos
- **Nomes Significativos**: Escolha nomes descritivos que revelem a intenção; evite abreviações e variáveis de uma única letra, exceto em contextos muito restritos
- **Funções Pequenas e Focadas**: Mantenha funções pequenas e focadas em uma única tarefa para melhor legibilidade e testabilidade
- **Indentação Consistente**: Use indentação consistente (espaços ou tabs) e configure seu editor/linter para aplicá-la
- **Remover Código Morto**: Delete código não utilizado, blocos comentados e importações desnecessárias em vez de deixá-los acumulando
- **Compatibilidade retroativa apenas quando necessário**: A menos que instruído especificamente, assuma que você não precisa escrever lógica adicional para lidar com compatibilidade retroativa
- **Princípio DRY**: Evite duplicação extraindo lógica comum para funções ou módulos reutilizáveis
